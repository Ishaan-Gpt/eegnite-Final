## 🔒 Anti-Indexing Protections (Triple-Layer)

This staging package includes **THREE layers** of protection to prevent search engine indexing:

### ✅ Layer 1: robots.txt
- **Location:** `/robots.txt`
- **Content:** Blocks ALL user agents (Google, Bing, Yahoo, DuckDuckGo, Baidu, Yandex, etc.)
- **Verification:** Visit `https://your-staging-domain.com/robots.txt`

### ✅ Layer 2: Meta Tags
- **Applied to:** All HTML pages
- **Tags:** `<meta name="robots" content="noindex, nofollow">`
- **Generated by:** Next.js layout (automatic)

### ✅ Layer 3: HTTP Headers
- **Header:** `X-Robots-Tag: noindex, nofollow, noarchive, nosnippet`
- **Applied via:** .htaccess file
- **Scope:** All pages and assets

## 🚀 Deployment Instructions (Hostinger)

### Step 1: Prepare Your Staging Environment

1. Log in to your Hostinger control panel
2. Navigate to **File Manager**
3. Go to your staging domain's `public_html` directory
4. **IMPORTANT:** Delete all existing files in the directory (or backup first)

### Step 2: Upload the Package

**Option A: Via File Manager (Recommended)**
1. In File Manager, click **Upload**
2. Upload `eegnite-staging-noindex.zip`
3. Right-click the uploaded ZIP file
4. Select **Extract**
5. Delete the ZIP file after extraction

**Option B: Via FTP**
1. Connect to your Hostinger via FTP (use FileZilla or similar)
2. Navigate to `public_html` (or your staging subdomain folder)
3. Upload ALL files from the extracted `out` folder
4. **CRITICAL:** Ensure `.htaccess` is uploaded (it may be hidden)
   - In FileZilla: Server > Force showing hidden files
   - Make sure the `.htaccess` file is visible and uploaded

### Step 3: Verify the Deployment

**Test 1: Website is Accessible**
- Visit: `https://your-staging-domain.com`
- The homepage should load correctly

**Test 2: robots.txt is Blocking Indexing**
- Visit: `https://your-staging-domain.com/robots.txt`
- Should show: "STAGING ENVIRONMENT - DO NOT INDEX"
- Should contain: `Disallow: /` for all user agents

**Test 3: HTTP Headers (Advanced)**
- Use browser DevTools (F12) > Network tab
- Refresh the page
- Click on the main document request
- Check Response Headers for:
  ```
  X-Robots-Tag: noindex, nofollow, noarchive, nosnippet
  ```

**Test 4: Staging Information Page**
- Visit: `https://your-staging-domain.com/STAGING_INFO.html`
- Should display a yellow warning box confirming staging configuration

**Test 5: Meta Tags**
- View page source (Ctrl+U)
- Search for: `<meta name="robots"`
- Should contain: `content="noindex, nofollow"`

## ⚡ Troubleshooting

### Issue: Pages show "404 Not Found"
**Solution:** 
- Ensure `.htaccess` was uploaded
- Check file permissions: .htaccess should be 644
- Confirm mod_rewrite is enabled (Hostinger has it by default)

### Issue: Trailing slashes causing errors
**Solution:**
- The `.htaccess` handles this automatically
- Next.js static export uses trailing slashes
- URLs should be: `/services/seo/` (with trailing slash)

### Issue: Images not loading
**Solution:**
- Check that the `_next` folder was uploaded completely
- Verify the `public` assets are in the root directory
- Check browser console for 404 errors

### Issue: .htaccess not working
**Solution:**
- Rename it to `htaccess.txt`, upload, then rename back to `.htaccess`
- Or use File Manager to create it directly
- Ensure it's in the root public_html directory

## 🔄 Moving to Production

When you're ready to deploy to production:

1. **DO NOT use this staging package**
2. Build a fresh production package:
   ```bash
   npm run build
   ```
3. Ensure production robots.txt allows indexing
4. Remove or replace the `.htaccess` to remove the X-Robots-Tag header
5. Set `NEXT_PUBLIC_SITE_URL=https://www.eegnite.com`
6. Deploy to your production domain
